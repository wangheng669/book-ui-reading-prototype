# Memory Interaction Rules V0.1

> 目标：验证用户能否从记忆中提取并重新组织内容，而不是用“看过页面”代替学习。记忆交互必须服从阅读节奏，先回忆、后提示、再回到原文。

## 一、状态定义

- **读过：** 用户到达并停留过知识点，只代表发生了接触；不能显示完成勾选。
- **理解：** 用户能用自己的话解释含义、条件或关系，或明确确认自己理解；仍不等于之后能够回忆。
- **能够回忆：** 在不查看提示和原文的情况下，用户能提取关键内容，并覆盖该知识点的必要维度。
- **真正完成：** 至少完成一次主动回忆，答案覆盖最低要点；或用户在看到差异反馈后明确确认“我已掌握”。滚动经过、展开组件、停留时长和重复阅读都不能自动完成节点。

## 场景 1：单个知识点结束后的轻量主动回忆

### 触发与不触发

- **触发时间：** 用户读完知识点并继续阅读一小段，在段落间、案例结束或小节转换等自然停顿处，显示一行轻量入口；不立即在知识点结尾贴题。
- **不触发：** 用户快速滚动、正在选词/批注/朗读、刚使用过其他辅助、已选择“稍后再说”、知识点过于简单、或系统无法确认其边界时。
- **避免打断：** 入口嵌入正文节奏，不遮挡、不自动聚焦、不改变正文宽度；用户忽略后继续阅读，系统不追问、不悬浮跟随。

### 回忆流程

1. 初始只显示一个问题与两个动作：**“用自己的话回答”**、**“稍后再说”**。
2. 用户主动选择回答后，才展开简洁输入区；默认不显示提示、标准答案或关键词。
3. 提交后再显示“你的回答 / 原文要点”差异反馈。反馈按关键维度标记为“已提及 / 可补充”，不用红绿、分数、对错叉号或人格化评价。
4. 若遗漏关键项，先给一个最小提示并允许补答；仍需要帮助时才展开原文要点。
5. 每个维度使用数字锚点返回对应原文句子；返回后保留用户答案，且不自动判为完成。

### 错误反馈与完成条件

- 错误或不完整回答应说明“缺少哪个关系或条件”，而不是只给正确答案。
- 允许用户选择“回到原文”“再答一次”或“标记稍后复习”。
- **完成：** 独立回答覆盖预设的最低要点；或用户查看差异、补充理解后明确确认掌握。
- **未完成：** 跳过、空答、直接看提示、仅返回原文或系统仅检测到已读。此时节点状态为“读过”或“需要复习”。

## 场景 2：章节结束后的结构化复盘

### 触发与不触发

- **触发时间：** 用户到达章节正文末尾，或之后从书籍地图主动进入；入口与“继续下一章”并列但不强迫完成。
- **不触发：** 用户尚未读到章节末、正在连续阅读、章节过短且无可回忆结构、或本次阅读只完成极少内容时。
- **避免打断：** 复盘是独立阅读状态，不覆盖最后一页正文；随时可“返回原文”或“稍后复盘”，退出后保留进度。

### 复盘顺序

1. **闭卷回忆优先：** 首屏只显示 3–5 个核心节点名称和 2–3 个回忆题，不显示结构摘要、标准答案或节点关系。
2. 用户作答或选择跳过后，才显示结构摘要，并把回答映射到节点。
3. 增加 1 个迁移题，要求把本章原则用于新的投资情境；迁移题不追求唯一措辞，反馈关注推理是否使用了正确原则。
4. 遗漏项按“需要复习”呈现，每项都有数字锚点返回原文；复习后可再次回忆。

### 节点状态与反馈

- **仅阅读过：** 到达过原文，但未独立解释或回忆。
- **能够解释：** 当前回答能说明概念或关系，但尚无延迟回忆证据。
- **需要复习：** 未回答、关键维度遗漏、依赖提示，或迁移时使用错误原则。
- 节点不可因滚动到章节末而自动勾选。完成勾选只在主动回忆达到最低要点，或用户明确确认掌握后出现。
- 反馈使用文字层级和蓝灰中性色，不使用红绿成绩、百分比、排名、连续打卡或总分。

## 共同约束

- 先回答，后提示；先指出差异，再展示完整答案。
- “返回原文”必须定位到具体句子或段落，使用短数字锚点，不使用跨屏长连线。
- 用户返回原文后仍可回到原问题，答案与遗漏项保持不丢失。
- 系统衡量的是提取与解释，不以阅读时长、滚动距离或组件打开次数替代记忆证据。
- 任何时刻都允许退出并继续阅读；记忆交互不能成为阅读通行证。
