# 第一章阅读交互原型

## 覆盖范围

- 章节：`第 1 章 · 投资与投机`
- 知识点：`投资与投机`、`安全边际`
- 数据：全部来自 `src/content.js` 的本地静态对象
- 运行：React + Vite 单页原型；无模型、数据库、账号或网络请求
- 公开地址：<https://wangheng669.github.io/book-ui-reading-prototype/>

## 状态流

### 阅读与知识点

`安静阅读 → 知识点进入视窗 → 轻量提示 → 用户主动展开辅助 → 数字锚点返回原文 → 离开范围后自动收起 → 安静阅读`

- IntersectionObserver 持续维护所有知识点的 intersectionRatio，并通过阈值、滞回和延迟退场稳定选择当前焦点。
- 滚动不会产生“能够解释”或“能够回忆”。
- 辅助组件打开状态不跨知识点保留。
- 开发环境右下角显示 activePoint、openAid、阅读状态、比例和知识点进度；生产部署不显示。

### 轻量回忆

`问题 + 两个操作 → 用户选择回答 → 仅展开输入框 → 提交 → 已提及/可补充 → 用户主动展开完整原文对照`

- 每个知识点各有一次轻量回忆。
- 提交答案覆盖至少两个预设维度后，节点进入“能够解释”。
- 完整原文对照使用原生 details，默认关闭。

### 章节复盘

`闭卷问题 → 完成或跳过 → 结构梳理 + 遗漏项 + 迁移题`

- 第一阶段不渲染结构摘要。
- 第二阶段展示能力状态：仅接触、能够解释、能够回忆。
- 建议复习是独立标签，不替代能力状态。
- 闭卷回答按定义、依据、收益来源、风险态度和安全边际作用分别计算 covered / missing。
- 同一知识点的最低必要维度全部 covered 后，才进入“能够回忆”；单题或单关键词不能完成知识点。
- 迁移题提交前不显示反馈，提交后按多信号原则显示“已使用 / 可补充”，并可返回对应原文。

## 手动测试

1. 从章节开头缓慢向下滚动，确认安静阅读时右侧为空；进入“投资与投机”后才出现轻量提示。
2. 在知识点上下边界小幅往返，开发面板中的 activePoint 不应短暂变为 null 或频繁切换。
3. 展开纵向辅助，点击原文数字锚点；离开知识点后辅助应自动收起。
4. 在轻量回忆中依次验证：初始问题、展开输入框、提交后的简短反馈、手动展开完整对照。
5. 滚动至“安全边际”，重复提示、辅助、锚点和回忆流程。
6. 进入章节复盘，确认第一阶段没有结构摘要；只填写“投资与投机”的第一题后完成，节点不应显示“能够回忆”。
7. 补齐同一知识点的全部必要维度，再次复盘时才应显示“能够回忆”。
8. 迁移题提交前不应显示原则提示；提交后检查“已使用 / 可补充”和原文锚点。
9. 打开生产地址，确认开发状态面板不显示。

## 静态内容数据结构

```js
chapter = {
  id,
  book,
  title,
  intro,
  knowledgePoints: [{
    id,
    title,
    shortPrompt,
    anchors: [{ id, label, investment?, speculation?, detail?, keywords }],
    paragraphs: [{ anchor, text }]
  }]
}

reviewQuestions = [{
  id,
  pointId,
  question,
  keywords
}]

transferQuestion = "..."
```

运行时状态按知识点保存：

```js
{
  exposure: boolean,  // 仅接触
  explained: boolean, // 能够解释
  recalled: boolean,  // 能够回忆
  review: boolean,    // 建议复习标签
  dimensions: { dimensionId: "covered" | "missing" }
}
```
