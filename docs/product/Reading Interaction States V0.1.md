# Reading Interaction States V0.1

> 目标：让辅助内容像阅读中的短暂注释，而不是常驻信息栏。大部分时间属于正文；系统只在关键知识点需要额外表达时介入。

## 1. 安静阅读

- **正文区域：** 保持完整宽度、稳定行长与连续段落；不高亮普通文字，不改变滚动节奏。
- **右侧辅助：** 不显示具体内容。可完全留白，或仅保留几乎不可见的结构占位，不能形成“等待填充的栏”。
- **出现与消失：** 无组件；此前组件应已淡出或收起。
- **动画：** 不需要。恢复安静状态时只允许 150–250ms 的透明度下降，不移动正文。
- **注意力：** 始终停留在正在阅读的行与下一段。
- **绝不弹出：** 普通叙述顺畅、用户连续滚动、刚关闭辅助、正在选择/批注文字、或系统无法确认内容价值时。

## 2. 即将进入关键知识点

- **正文区域：** 目标段落旁出现极轻的定位标记，例如短竖线或小型知识节点；不整段染色，不改变字号和行距。
- **右侧辅助：** 只出现低对比度标题或轮廓，提示“这里有一项可选辅助”，不提前展示答案。
- **出现与消失：** 当关键段落进入阅读视窗的中部区域时渐显；若用户快速越过，立即取消，不追赶弹出。
- **动画：** 允许 180–280ms 淡入；禁止滑入、弹跳、闪烁和放大。
- **注意力：** 仍在正文。右侧只可被余光感知，不能要求扫视。
- **绝不弹出：** 用户尚未读到相关前提、正在快速滚动、知识点可在正文内用一句轻注表达、或右侧内容会泄露论证结论时。

## 3. 当前关键知识点激活

- **正文区域：** 相关的 2–3 段通过细线、低饱和底色或段侧锚点建立范围；关键句只做一种强调。
- **右侧辅助：** 可显示一个与当前知识点匹配的组件；必须明确标注对应正文范围，并保持低于正文的视觉权重。
- **出现与消失：** 当对应段落稳定停留在阅读焦点区后出现；离开范围即降级，不跨知识点常驻。
- **动画：** 只允许透明度与极小位移的 200–300ms 过渡；正文不得重排。
- **注意力：** 默认仍在正文。辅助负责“看一眼即懂”的关系表达，而不是另一篇文章。
- **绝不弹出：** 同屏已有批注、选词解释或其他焦点；组件不能增加理解；用户已明确忽略或关闭该知识点辅助时。

## 4. 用户主动深入查看

- **正文区域：** 保持当前位置，不被遮挡或缩窄；对应段落锚点增强，点击辅助项可回到具体原文句子。
- **右侧辅助：** 完整展开当前组件，可显示必要细节与“定位原文”；不出现第二个组件、推荐问题或聊天入口。
- **出现与消失：** 只由点击、键盘操作或明确选择触发；关闭后返回“当前关键知识点激活”，而非切换页面。
- **动画：** 允许 200–320ms 展开/折叠与内容交叉淡化；必须支持减少动态效果。
- **注意力：** 可短暂转向辅助区，但每一项都应能回接原文；完成后自然返回正文。
- **绝不弹出：** 未经用户主动操作；用户正在输入笔记、复制原文或使用屏幕阅读器连续朗读时。

## 5. 离开知识点或进入章节回顾

- **正文区域：** 知识点标记降回普通节点，正文恢复安静排版；章节末才允许出现回顾入口。
- **右侧辅助：** 离开知识点后淡出、收起为极弱标记或完全消失；下一知识点只提示存在，不提前展示内容。
- **出现与消失：** 滚动越过对应范围后在 200–350ms 内降级；章节回顾只在正文结束或用户主动进入时出现。
- **动画：** 允许克制的淡出、折叠；禁止组件追随滚动、突然换成下一张卡或连续轮播。
- **注意力：** 回到后续正文；章节结束时才转向主动回忆与结构复盘。
- **绝不弹出：** 用户刚离开知识点、快速浏览后文、尚未到章节结尾，或回顾会提前暴露下一段答案时。

## 状态转换约束

`安静阅读 → 即将进入 → 知识点激活 →（用户主动）深入查看 → 激活/离开 → 安静阅读或章节回顾`

- “深入查看”只能由用户主动触发；其余状态由阅读位置与停留行为共同决定。
- 任何自动状态都可直接退回“安静阅读”；系统不应为了展示组件而阻止退场。
- 同屏最多一个激活知识点。能在正文内轻量表达的内容，不迁移到右侧。
- 若无法可靠判断用户正在阅读哪个知识点，默认进入“安静阅读”。
